## Collaboration Process

- At the start of active research sessions, run `make codex-start`.
- For micro tasks, record a quick event/snapshot only when useful. At standard/deep closeout, update the daily log, run `make codex-sweep` when files changed, and update current context. Use `make codex-close-fast`, `make codex-close-standard`, or `make codex-close-deep` according to task weight.
- Update weekly review once per active research week.
- When adding or changing workflow functions, also update a user-visible entry, guide, or project board so the user can see what was built and how to use it.
- For paper co-reading, start from the context pack when available; open the full Reader only when the user needs source checking, page-level detail, or deeper paragraph-level reading.
