# 图书馆短视频研究：创新点、局限性与机会台账

Project: `library_short_video`
Created: 2026-06-20
Purpose: 把每天主读论文中的创新点、局限性和可转化研究机会持续沉淀下来，后续用于选题、研究设计、论文讨论和改进方案生成。

## 使用规则

- 只记录已经 source-grounded 阅读过的论文；未读元数据不得写成稳定判断。
- 每条创新点和局限性都要保留证据定位，优先使用 reader block ID；后续人工核页后再补页码。
- “机会”不是直接等于选题，必须写清它从哪类创新或局限转化而来。
- 每次主读结束后至少补三类内容：可复用创新、关键局限、可转化研究问题。
- 写论文时仍需回到原文和 evidence gate，不把本台账当作最终证据来源。

## 分类体系

### 创新点类型

| 类型 | 含义 | 后续用途 |
|---|---|---|
| 理论/模型引入 | 把外部理论或传播模型用于图书馆短视频/阅读推广问题 | 构建研究框架、提出变量路径 |
| 对象/场景拓展 | 研究对象从一般短视频、政务号、社交号转入图书馆或阅读推广场景 | 判断研究边界和可迁移性 |
| 数据/样本建设 | 形成可复用的平台样本、账号样本或时段样本 | 设计复现、扩展或比较研究 |
| 变量/测量设计 | 把内容、标题、封面、时长、音乐等转成可分析指标 | 形成编码表、变量字典、指标体系 |
| 方法/技术路径 | 使用统计检验、回归、文本挖掘、图像识别等方法 | 提升方法严谨性和可自动化程度 |
| 实践策略转译 | 把实证结果转成图书馆运营建议 | 对接实践贡献与应用场景 |

### 局限性类型

| 类型 | 含义 | 后续可转化方向 |
|---|---|---|
| 指标替代问题 | 点赞、评论、分享等平台互动不能直接代表图书馆服务价值 | 构造“传播效果 + 服务价值”复合指标 |
| 样本外推问题 | 样本平台、账号类型、时间窗口有限 | 做跨平台、跨馆型、跨时段比较 |
| 因果识别问题 | 观察性数据只能说明相关或差异，不能直接证明因果 | 加入准实验、时间序列、A/B 或匹配设计 |
| 变量遗漏问题 | 音乐、剪辑、主持人、主题、算法推荐等变量未充分量化 | 引入多模态特征和更细编码 |
| 平台时代漂移 | 早期抖音规则、内容生态与当前平台环境不同 | 更新到 2024-2026 数据重新检验 |
| 图书馆转化不足 | 外部社交号或商业号经验不能直接等同于图书馆实践 | 建立图书馆专属样本和实践验证 |

- RQ-002: Hook Model 中的触发、行动、酬赏、投入四类机制，在公共馆、高校馆和科研型图书馆中是否存在差异？
- RQ-003: 早期研究发现的标题句型、音乐、制作类别、内容主题、信息类型等显著因素，在 2024-2026 年抖音/视频号/B站环境下是否仍然显著？
- RQ-004: 图书馆短视频中“有用信息酬赏”是否比情绪触发更能解释服务转化，而不仅是点赞转发？

- RQ-006: 图书馆阅读推广短视频的中心线索（内容质量、可读性、主题）与边缘线索（标题、封面、时长、信源）哪一类更能解释服务价值指标？
