Hook Model 上瘾模型

一句话解释

Hook Model 是一个解释“用户为什么会一次又一次回来使用某个产品或内容”的行为循环模型。

通俗解释

它可以理解成一个四步循环：

  1. 触发：什么东西提醒用户开始行动。
  2. 行动：用户做了一个很容易完成的小动作。
  3. 多变酬赏：用户得到一点有用、有趣、惊喜或情绪满足的回报。
  4. 投入：用户留下了一点成本或关系，使下一次更可能回来。

这个循环如果不断重复，就会让用户形成习惯。

四个环节

环节简单说法图书馆短视频例子
Trigger 触发让我注意到它标题、封面、推送、热点、馆内活动提醒
Action 行动我点一下就能做点开视频、看完 30 秒、点赞、评论、收藏
Variable Reward 多变酬赏我得到了点什么，而且每次不完全一样学到一个冷知识、发现一本书、被故事打动、看到有趣馆员
Investment 投入我留下了一点关系或成本关注账号、收藏书单、评论提问、报名活动、转发给同学

生活例子

短视频平台就是典型例子：

  • 触发：你看到一个标题“这个冷知识你肯定不知道”。
  • 行动：你点进去看。
  • 多变酬赏：有时真的有用，有时很好笑，有时情绪被打动。
  • 投入：你点赞、收藏、关注账号。

下次平台再推类似内容，你就更容易继续点开。

图书馆短视频研究例子

如果图书馆发布一条短视频：

“3 分钟带你找到毕业论文最需要的数据库”

可以用 Hook Model 这样分析：

  • 触发：标题直接击中毕业论文需求。
  • 行动：用户点开并看完短视频。
  • 多变酬赏：用户发现一个以前不知道的数据库检索技巧。
  • 投入：用户收藏视频、关注图书馆账号，甚至预约数据库培训。

如果后续图书馆持续发布“论文选题、数据库检索、文献管理、引用规范”系列内容，用户就可能形成稳定关注。

为什么它对我的研究重要

在 library_short_video 项目中，Hook Model 可以帮助解释：

  • 为什么有些图书馆短视频能触发用户观看。
  • 为什么有些内容能带来点赞、评论、转发。
  • 为什么“服务型内容”可能比单纯娱乐内容更适合图书馆。
  • 如何从平台互动进一步思考服务转化。

它适合连接 图书馆短视频相关研究、阅读推广、服务价值 和 平台互动指标。

容易误解的地方

  • Hook Model 不是简单教人“让用户上瘾”，在图书馆研究里更应理解为“形成有价值的使用习惯”。
  • 高互动不一定等于高服务价值。用户点赞了，不代表真的使用了图书馆服务。
  • Hook Model 解释的是行为循环，不自动证明因果关系。
  • 图书馆不能只追求流量，还要考虑公共文化服务、阅读推广和知识服务使命。

我自己的理解

Hook Model 适合问：图书馆短视频怎样让用户从“看一眼”走向“持续关注和使用服务”。

复习问题

  1. Hook Model 的四个环节是什么？
  2. 为什么“点赞”只能算 Action 或 Reward 的结果之一，不能直接等于服务价值？
  3. 如果设计一条图书馆数据库推广短视频，四个环节可以分别是什么？

研究灵感

  • 比较图书馆短视频中的“流量型 Hook”和“服务型 Hook”。
  • 把 Hook Model 和 ELM 结合，分析用户是被标题/封面触发，还是被内容质量说服。
