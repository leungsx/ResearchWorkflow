# 文献综述工作台

Last updated: 2026-06-28

这个工作台把短视频提示词里的思路转成可复用的科研工作流。核心目标不是让模型直接代写一段长文献综述，而是先完成四层整理：

1. 逐篇文献主要结论汇总。
2. 按研究对象、机制、方法或问题意识进行逻辑分类。
3. 提炼适合写进文献综述的研究线索。
4. 明确哪些信息不能写、哪些需要补原文或页码。

## 输入边界

优先使用已经完成 source-grounded 阅读的材料：

- Reader Reading Notes
- 论文上下文包
- 研讨卡
- 创新-局限-机会台账
- 文献矩阵中的元数据
- Evidence Gate 报告

不能把 metadata-only、abstract-only 或 AI-summary-only 文献当成论文主张证据。它们只能用于候选池、检索地图或待补阅读清单。

## 单篇文献提取模板

每篇文献先按这个表整理，避免只复述摘要：

| 文献 | 研究主题 | 研究对象/数据 | 理论/方法 | 主要结论 | 与本研究主题的关系 | 可放入哪一类文献 | 证据状态 |
|---|---|---|---|---|---|---|---|
|  |  |  |  | 用 2-4 句精准概括，说明文章真正证明或主张了什么 | 直接相关 / 间接参照 / 方法模板 / 背景材料 / 待排除 | 由内容判断，不按年份机械分类 | metadata-only / skimmed / human-read / verified |

检查规则：

- “主要结论”不要复制摘要。
- “关系”必须说明这篇文献是在直接研究本主题，还是只提供外部参照。
- “类别”根据文章内容和论证功能判断，不按作者或年份机械排列。
- 如果信息不足，写“需要补充原文信息”，不要补造作者、年份、数据或结论。

## 跨文献逻辑分类模板

把文献分成 3-5 类即可。每一类都要回答共同点、差异和不足。

| 类别 | 包含文献 | 这一类主要讨论 | 共同结论 | 关键差异 | 这一类文献的不足 | 可自然引出的后续问题 |
|---|---|---|---|---|---|---|
| 类别一： |  |  |  |  |  |  |
| 类别二： |  |  |  |  |  |  |
| 类别三： |  |  |  |  |  |  |

分类建议：

- 可以按研究对象分：公共图书馆、高校图书馆、社会化阅读账号、平台用户。
- 可以按机制分：内容设计、互动传播、服务转化、平台适配。
- 可以按方法分：现状调查、内容分析、统计检验、回归建模、案例/策略归纳。
- 可以按论文功能分：背景诊断、理论模型、方法模板、反面提醒、实践策略。

## 综述线索生成模板

不要按“作者 A 说、作者 B 说”的顺序罗列。先给研究线索，再把文献放进去。

```text
已有研究大致形成了若干线索。

第一类研究关注……这类研究说明……但主要局限在……
第二类研究转向……它们进一步表明……但仍然缺少……
第三类研究尝试……这一线索的价值在于……不足在于……

总体来看，现有研究已经说明了……，但在……方面仍然不足。
因此，后续研究可以进一步关注……
```

## 禁忌

- 不要直接让模型替你写成长篇文献综述正文。
- 不要编造没有提供的文献、作者、年份、样本或数据。
- 不要把“传播效果”等同于“图书馆服务价值”，除非有额外证据。
- 不要把只读过摘要或元数据的文献写成已验证证据。
- 不要按作者年份机械排列文献，要说明研究之间的递进、互补、分歧和空白。
- 如果某篇文章与主题关系弱，直接标明，不要硬塞进主线。

## 可直接对 Codex 说

```text

```

```text

```
